指定されたPR番号のPRに対応するissueのドラフトをMarkdownで作成してください。

## 手順

1. **PR情報の取得**
   - PR番号が指定されている場合は `gh pr view {num} --json title,body,number,headRefName` でPRの情報を取得
   - PR番号が指定されていない場合は `gh pr view --json title,body,number,headRefName` で現在のPRの情報を取得
   - PRが存在しない場合はエラーメッセージを表示して終了

2. **issue内容の作成**
   以下の構成でissueのドラフトを作成:

   ```markdown
   ## 概要

   [PRのタイトルを元に、これから取り組む作業として簡潔に記述]

   ## 背景

   [PRの本文から背景・目的を抽出。なぜこの作業が必要なのかを説明]

   ## 完了条件

   - [ ] [PRの変更内容から完了条件を抽出]
   - [ ] [複数ある場合は箇条書きで列挙]

   ## 関連

   - [関連issueがあればリンク]
   ```

3. **出力**
   - 作成したMarkdownをそのまま表示
   - 実装詳細には触れず、課題と目的を中心に記述
   - これから取り組むという前提で書く（過去形は使わない）

## 注意事項

- PRの本文に既にissueがリンクされている場合は、関連issueとして記載
- 技術的な実装詳細よりも「何を達成するか」を重視
- 完了条件は検証可能な形で記述
